# 진단 스크립트 공통 양식·스타일 가이드

KISA 자동 진단 스크립트(WAS / WEB / Linux(UNIX) / DB / Windows) 공통 규칙.
**목표 파이프라인:** 로우데이터(CSV) → LLM 판정 → 대시보드 + 엑셀 보고서.
→ 점검 내용은 *가공 문장*이 아니라 **원문 증적**이어야 신뢰된다. 새 스크립트(DB·Windows 등)도 이 문서를 따른다.

> bash 계열(was/web/linux)은 아래 코드를 그대로 복사해 쓰고, PowerShell(Windows)은 같은 *출력 양식/원칙*을 동일하게 재현한다.

---

## 1. 파일 구성

```
deploy/<area>-diag/
  <area>_diag.sh            # 진단 본체 (읽기 전용)
  <area>_diag.conf          # 환경 의존값(경로·계정·임계값). 호스트별로 분리 가능
  <area>_diag_<host>.conf   # 호스트별 conf (커널/계정 등 다르면)
  result_<target>/          # 산출물(raw CSV + report TXT)
  README.md                 # 사용법
```
- 출력 2종: **로우데이터 `<area>_diag_raw_<label>_<ts>.csv`** + **보고서 `<area>_diag_report_<label>_<ts>.txt`**
- conf는 **선언 = 사용** (스크립트가 안 쓰는 변수 선언 금지 — 죽은 변수 ❌).

---

## 2. 출력 양식 (항목 블록)

```
[<코드> (<중요도>) <항목명>]
점검 상태    : 양호|취약|N/A          ← 항목명 바로 밑
점검 파일 명 : <경로1> | <경로2> | ...  ← 여러 개는 " | " 로 구분 (슬래시 ❌)
점검 내용    :
    <원문 증적 (들여쓰기 4칸)>
점검 요약    : <KISA 판단기준 문구 + (▸ 근거)>
----------------------------------------------------------------
```
- **점검 상태**는 항목명 바로 다음 줄(맨 위).
- **점검 파일 명**의 다중 경로 구분자는 ` | ` (경로 안의 `/`·`find /` 명령·주석은 그대로).
- 파일 경로는 **변수 사용**(`$SERVER_XML`), 리터럴 문자열("server.xml") ❌.

```bash
emit_block(){
  printf '[%s (%s) %s]\n' "$1" "$2" "$3"          # 코드 중요도 항목명
  printf '점검 상태    : %s\n' "$6"
  printf '점검 파일 명 : %s\n' "$4"
  printf '점검 내용    :\n'
  if [ -n "$5" ]; then printf '%s\n' "$5" | sed 's/^/    /'; else printf '    (없음)\n'; fi
  printf '점검 요약    : %s\n' "$7"
  printf -- '----------------------------------------------------------------\n'
}
```

사전 정보 헤더(보고서 상단)에 OS·IP·대상·**점검 시각(KST)**·기준·설정파일 표기.

## 3. 점검 내용(raw) 작성 원칙  ★핵심★

1. **파일에 적힌 값 = grep 원문 라인 그대로.** 추출값을 문장으로 풀어쓰지 않는다.
   - ❌ `세션 타임아웃: 600` / `암호화: YESCRYPT` / `maxPostSize="5242880`(조각)
   - ✅ `/etc/profile:TMOUT=600` / `ENCRYPT_METHOD SHA512` / `# server.xml: maxPostSize="5242880"`(전체 라인)
2. **여러 파일 참조 시 출처를 `# <파일경로>` 헤더로** 구분하고 그 아래 실제 라인.
   ```
   # /etc/login.defs
   PASS_MAX_DAYS 90
   # /etc/security/pwquality.conf
   minlen = 8
   ```
3. **정규식 조각 금지** — 값만 `-oE`로 잘라 따옴표가 깨지면 안 됨. 값은 *판정용*으로만 추출하고, **표기는 전체 라인**.
4. **값이 아예 없을 때만 서술형** `(…없음)` 허용.
5. **런타임 상태**(서비스 구동 여부 등 파일값이 아닌 것)는 상태 표기 OK: `대상 서비스(...): 구동중=[없음]`.
6. **권한은 stat 표현**으로: `-rw-r----- (640) tomcat:tomcat  /path` (`stat_line`).
7. 라인 정리: 앞 들여쓰기/탭 제거(`sed 's/^[[:space:]]*//' | tr -s ' \t' ' '`), 누적 문자열은 **앞/뒤 공백 트림**.

---

## 4. 판정 규칙

- 상태는 **양호 / 취약 / N/A** 3종만.
- **"무조건 양호" 스텁 금지** — 모든 항목은 실제 판정 분기를 가진다.
- **권한 비교는 칸별 부분집합 `perm_subset`** 사용. 8진수를 통째 정수로 비교(`perm_le`)하면 owner/group/other가 뭉개져 오판하므로 **금지**.
  - 예: 비번파일 `060`/`044`는 600의 부분집합이 아니므로 취약(통째 비교는 양호로 오판).
- **소유자·권한 실패 사유는 분리** 표기: `소유자 부적정(...)` vs `권한 과다(640 — 기준 400 부분집합 아님)`.
- **열거형 항목**(SUID/SGID, 숨김 파일 등 결과가 N개 목록):
  - 필터 없이 **전부 `stat`로 나열 + 총 개수**, 상태 **N/A(수동 확인 대상)**.
  - 과다 출력 방지로 상한(예 100) + `... 외 M개` 표기, 전체는 raw CSV 참조.
- N/A = 미해당(서비스 미구동 등) 또는 자동 판정 불가(수동 확인). 어느 쪽인지 요약에 명시.
- 요약 문구는 **KISA 판단기준(CRIT_PASS/VULN)** 에 직접 대응시키고 `(▸ 실제 근거)` 덧붙임.

## 5. 공통 헬퍼 (그대로 복사)

```bash
perm_of(){ [ -e "$1" ] && stat -L -c '%a' "$1" 2>/dev/null || echo ""; }
owner_of(){ [ -e "$1" ] && stat -L -c '%U:%G' "$1" 2>/dev/null || echo ""; }
stat_line(){ [ -e "$1" ] && stat -L -c '%A (%a) %U:%G  %n' "$1" 2>/dev/null; }
# 파일 권한이 기준 권한의 '칸별 부분집합'이면 통과(owner/group/other 비트별)
perm_subset(){ [ -n "$1" ] && [ "$(( (8#$1) & ~(8#$2) ))" -eq 0 ]; }
others_has_access(){ local p="$1"; [ -n "$p" ] && [ "$(( 8#$p % 8 ))" -ne 0 ]; }
others_has_write(){  local p="$1"; [ -n "$p" ] && [ "$(( (8#$p % 8) & 2 ))" -ne 0 ]; }
group_has_write(){   local p="$1"; [ -n "$p" ] && [ "$(( ((8#$p/8) % 8) & 2 ))" -ne 0 ]; }
```

용도: 권한 기준비교는 `perm_subset`, "other/그룹 접근 유무"는 `others_has_*`/`group_has_*`.
(파일별 기준이 다른 일반 권한 점검엔 일괄 `others_*` 가드 대신 `perm_subset`가 정답 — passwd 644의 other 읽기는 정상, shadow 400은 금지처럼 칸별로 갈리므로.)

## 6. conf 규칙

- 경로·계정·임계값 등 **환경 의존값만** conf에. 스크립트는 conf 변수를 source해서 사용.
- **죽은 변수 금지**: 선언했으면 스크립트가 실제로 써야 한다(conf↔script 일관).
- 호스트마다 다른 값(커널 버전, 시스템 계정 목록 등)은 **호스트별 conf**로 분리.
- `set -u` 환경에선 스크립트가 참조하는 변수를 **모든 conf가 반드시 정의**(미정의 시 중단). 선택값은 `${VAR:-기본값}`.
- 권한 기준 주석은 "이하" 같은 표현 대신 **"칸별 부분집합"** 의미로 기술(perm_subset 사용 시).

## 7. 로우데이터(CSV)

- **UTF-8 BOM** 선두 출력(Excel 한글 깨짐 방지): `printf '\xEF\xBB\xBF'`
- **1항목 = 1행.** 컬럼: 항목코드·분류·중요도·점검항목·점검파일·대상·결과·점검내용·점검요약·진단대상·점검시각.
- 멀티라인 점검내용은 한 셀에 ` | ` 로 조인(아래 `csv_field`).
- 열거형(목록 큰 항목)은 추후 **상세 CSV 분리**(항목코드 ↔ 개별 엔트리) 가능 — 대시보드/엑셀 단계에서.

```bash
csv_field(){ local v; v="$(printf '%s' "$1" | sed 's/"/""/g' \
  | awk '{a[NR]=$0} END{for(i=1;i<=NR;i++) printf "%s%s",(i>1?" | ":""),a[i]}')"; printf '"%s"' "$v"; }
```

## 8. 환경 / 실행 규칙

- **읽기 전용(READ-ONLY)**: cat/grep/stat/ls/ps/find/systemctl(상태조회)만. 설정 변경 ❌.
- **타임스탬프 KST 고정**: 스크립트 상단 `export TZ='Asia/Seoul'` (서버가 UTC여도 점검시각·파일명 KST).
  - PowerShell: `[TimeZoneInfo]::ConvertTimeBySystemTimeZoneId([DateTime]::UtcNow,'Korea Standard Time')`
  - PS5.1 한글 .ps1은 **UTF-8 BOM 필수**, 결과는 `Out-File -Encoding utf8`(5.1은 BOM 포함).
- **권한 사전 점검**: root/관리자 권한 필요 자원(shadow 등)을 읽어야 하면 비-root는 안내 후 `exit 2`.
- `set -u`(미정의 변수 사용 방지). 큰 변경 전 의도 확인.

---

## 9. 항목 범위 원칙

- **항목 이관 금지**: 각 점검은 자기 코드(WEB-NN/U-NN 등) 안에서 처리. "이건 OS 영역"식 다른 진단으로 이동 ❌.
- **항목이 요구하지 않는 분석 금지**: 판단기준(CRIT)에 직접 대응하는 수집만. 과잉 콘텐츠 해석(평문/암호화/주석원문 등) ❌.
- 판단: "이 데이터가 해당 항목의 CRIT_PASS/VULN 문구에 직접 대응하는가?" → 그렇다면 수집.

---

## 10. 표현·문구 체크리스트 (커밋 전)

- [ ] 파일값을 문장으로 풀어쓰지 않고 **원문 라인**으로 표기했는가
- [ ] 다중 파일은 `# <경로>` 헤더로 출처 구분했는가
- [ ] 점검 파일 명 다중 구분자가 ` | ` 인가 (리터럴 경로 ❌, `$VAR` 사용)
- [ ] 누적 문자열의 **앞/뒤 공백** 트림했는가 (`"$(echo $found)"` / `${v# }` / `sed 's/ *$//'`)
- [ ] 별칭 **중복**(chronyd/chrony 등) 제거했는가
- [ ] 권한 비교가 `perm_subset` 인가 (`perm_le` ❌)
- [ ] "무조건 양호" 스텁이 아닌 실제 판정인가
- [ ] 열거형은 전수 나열 + count + N/A 인가
- [ ] BOM·KST·읽기전용·set -u 적용했는가
