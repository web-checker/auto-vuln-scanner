[ 2026 주요정보통신기반시설 기술적 취약점 분석·평가 상세가이드 - 웹 서비스(Tomcat) 조치사례 오류 정리 ]
대상: Apache Tomcat. 가이드 조치사례의 파일 위치/파일명/요소 오류만 정리.

================================================================

WEB-1 : Default 관리자 계정명 변경
원본 위치 : <Tomcat 설치 디렉터리>/conf/server.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/tomcat-users.xml
틀린 이유 : <user username="admin" ... roles="manager-gui"/>는 서버 설정이 아니라 Tomcat 관리자 계정/권한 설정이므로 server.xml이 아닌 tomcat-users.xml에 작성해야 함.

WEB-2 : 취약한 비밀번호 사용 제한
원본 위치 : <Tomcat 설치 디렉터리>/conf/server.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/tomcat-users.xml
틀린 이유 : WEB-01과 동일. <user username="admin" password="..." roles="manager-gui"/>의 비밀번호는 관리자 계정 설정이므로 server.xml이 아니라 tomcat-users.xml의 user 요소에 작성해야 함.

WEB-3 : 비밀번호 파일 권한 관리
원본 위치 : <Tomcat 설치 디렉터리>/tomcat-users.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/tomcat-users.xml
틀린 이유 : tomcat-users.xml은 conf 디렉터리에 위치함. chmod 600 대상 경로에 conf/ 가 누락됨.

WEB-4 : 웹 서비스 디렉터리 리스팅 방지 설정
원본 위치 : <Tomcat 설치 디렉터리>/web.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/web.xml
틀린 이유 : listings 옵션(DefaultServlet)이 들어가는 전역 web.xml은 conf 디렉터리에 위치함. 경로에 conf/ 가 누락됨.

WEB-8 : 웹 서비스 파일 업로드 및 다운로드 용량 제한
원본 위치 : <Tomcat 설치 디렉터리>/conf/web.lxml
수정 위치 : <Tomcat 설치 디렉터리>/conf/web.xml
틀린 이유 : "web.lxml"은 오타. 올바른 파일명은 web.xml.

WEB-11 : 웹 서비스 경로 설정
원본 위치 : web.xml (가이드 표기: "web.xml 파일 내 docBase를 별도의 경로로 변경")
수정 위치 : <Tomcat 설치 디렉터리>/conf/server.xml (또는 conf/context.xml)
틀린 이유 : <Host>·<Context docBase>는 web.xml 요소가 아님. <Host>는 server.xml에만 존재하고, Context(docBase)는 server.xml 또는 context.xml에 위치함. 같은 가이드의 WEB-06·WEB-17은 동일 요소를 server.xml로 올바르게 표기하고 있어 WEB-11만 모순됨.

WEB-12 : 웹 서비스 링크 사용 금지
원본 위치 : <Context allowLiking="true">
수정 위치 : <Context allowLinking="false"> (또는 allowLinking 옵션 제거)
틀린 이유 : ① "allowLiking"은 오타 → "allowLinking"(n 누락). ② '링크 사용 금지' 항목인데 조치 예시가 allowLinking="true"(허용=취약 상태)로 되어 있음. 양호 조치는 false 설정 또는 옵션 제거여야 함.

WEB-17 : 웹 서비스 가상 디렉터리 삭제
원본 위치 : <Tomcat 설치 디렉터리>/server.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/server.xml
틀린 이유 : server.xml은 conf 디렉터리에 위치함. 경로에 conf/ 가 누락됨.

WEB-19 : 웹 서비스 SSI(Server Side Includes) 사용 제한
원본 위치 : <Tomcat 설치 디렉터리>/tomcat-users.xml  (가이드: cat .../tomcat-users.xml | grep 'SSIServlet\|SSIFilter')
수정 위치 : <Tomcat 설치 디렉터리>/conf/web.xml
틀린 이유 : SSIServlet의 servlet-mapping, SSIFilter의 filter-mapping은 web.xml에 정의됨. tomcat-users.xml(계정 파일)에는 SSI 설정이 존재하지 않으므로 grep 대상 파일이 잘못됨.

WEB-24 : 별도의 업로드 경로 사용 및 권한 설정
원본 위치 : <Tomcat 설치 디렉터리>/conf/context.xml
수정 위치 : <Tomcat 설치 디렉터리>/conf/web.xml
틀린 이유 : 예시의 <servlet><servlet-name>fileUploadServlet</servlet-name><servlet-class>...</servlet-class></servlet>은 web.xml 요소임. context.xml에는 <servlet> 정의가 들어가지 않음. (덧붙여 Step 1 제목 "Context 요소 allowLinking 옵션 설정"도 업로드 경로 항목과 무관한 복붙 오류)

================================================================
[ 참고 - 파일 위치 오류는 아니나 기재상 문제 ]

WEB-6 : 웹 서비스 상위 디렉터리 접근 제한 설정
- 파일 위치(conf/server.xml)는 정확하나, "상위 디렉터리 접근(../)" 항목의 조치로 allowLinking(심볼릭 링크 옵션)을 제시 → 개념 불일치(allowLinking은 WEB-12 링크 항목 주제). 또한 Tomcat 8/9에서는 allowLinking이 Context 속성이 아니라 <Resources allowLinking="true"/> 중첩 요소로 변경됨.

WEB-9 : 웹 서비스 프로세스 권한 제한
- chown 예시 경로가 "/[Tomcat 설치 디렉터리]/usr/share/tomcat9/" 처럼 설치 디렉터리 변수와 절대경로(/usr/share/...)를 혼용 → 경로 표기 불일치.

WEB-12 / WEB-16 등 : "Step 1)"이 연속 중복 표기되는 단계 번호 오류 존재(일부 항목).
